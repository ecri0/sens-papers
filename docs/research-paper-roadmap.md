## 4. 起草の原則（系統 B、確定）

1. 実証データを必ず添える（推測と実証を明確区別）
2. 対立仮説を並列議論（必然性 vs 偶然性 / 客観性 vs 主観性 / 一意性 vs 多様性）
3. 限界を明示（検証範囲・データ偏り・一般化制約）
4. 既存研究との接続（6つの知的基盤 = 複雑系自己組織化 / Free Energy Principle /
   Predictive Coding / Universal Approximation / 情報理論的最適性 / 言語進化の構造的制約）
5. 思想的展開を抑制（哲学的含意は Position Paper か Manifesto へ分離）
